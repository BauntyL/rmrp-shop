Промт для создания сайта RMRP SHOP (уточнённый)
Название: RMRP SHOP

Цель сайта: Торговая площадка для продажи игровых авто, недвижимости, рыбы и кладов.

Основные требования:
1. Интерфейс и дизайн
Современный, чистый, удобный и адаптивный дизайн.

Главная страница с крупными баннерами и навигацией по категориям.

Удобное меню с основными разделами: Авто, Недвижимость, Рыба, Клады, Избранное, Мои товары, Диалоги.

Карточки товаров с фото, кратким описанием, ценой и кнопкой «Добавить в избранное» (сердечко).

Модальные окна и всплывающие подсказки для интерактивных элементов.

Цветовая схема приятная для глаз, с акцентами на важные кнопки.

2. Функционал пользователя
Регистрация и авторизация:

Отдельные страницы для регистрации и для авторизации.

Страница регистрации с формой: Имя, Фамилия, Пароль.

Страница входа с формой логина и пароля.

Создание объявления:

Пользователь выбирает категорию (Авто, Недвижимость, Рыба, Клады).

Заполняет простую форму с обязательными полями (например, для авто: марка, модель, тип авто и т.д.).

Отправляет объявление на модерацию.

Модерация:

Объявления попадают на проверку модераторам и администраторам.

После одобрения — объявление появляется в соответствующей категории.

Избранное:

Пользователь может добавлять товары в избранное, нажимая на иконку сердечка.

Все избранные товары отображаются во вкладке «Избранное».

Мои товары:

Отображаются все объявления пользователя.

Пользователь может снять объявление с продажи.

Каждое объявление привязано к одному из серверов.

5. Диалоги между пользователями
Личный чат между пользователями.

Сообщения модерируются модераторами и администраторами.

Возможность просмотра списка диалогов и сообщений.

6. Административный функционал
Управление пользователями:

Изменение имени пользователя.

Назначение ролей (пользователь, модератор, администратор).

Бан пользователя (на время или навсегда).

Модерация объявлений и сообщений в диалогах.

7. Технологии и деплой
Frontend: React или Vue с UI-фреймворком (например, TailwindCSS или Material UI).

Backend: Node.js/Express или Django/Flask для API и логики.

База данных: PostgreSQL на Railway для хранения пользователей, объявлений, диалогов и избранного.

Хостинг: Размещение фронтенда и бэкенда на Railway, исходный код на GitHub.
